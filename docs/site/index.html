1. Что делает Neo Link

Neo Link подключает ваш Telegram-бот к системе продажи трафика. Вы добавляете бота по токену, получаете API-ключ, проходите модерацию, а затем встраиваете в своего бота логику работы со спонсорами.

Цена для клиента сервиса:
• с проверкой: 2.5 ₽
• без проверки: 1.5 ₽

Доход владельца подключённого бота:
• 1 ₽ за подтверждённую подписку

2. Как подключить бота

  1. Откройте раздел Продажа трафика.
  2. Нажмите Добавить нового бота.
  3. Отправьте токен Telegram-бота.
  4. Neo Link проверит токен через getMe.
  5. После проверки вы получите API key.
  6. Заявка уйдёт на модерацию.

Статусы бота

  • 🟡 На модерации — бот ещё не допущен к работе.
  • 🟢 Допущено — бот одобрен и может работать.
  • 🔴 Отклонено — бот отклонён модерацией.
  • ⏸ Остановлен — бот выключен владельцем и временно не участвует в логике API.

3. Как устроен поток работы

  1. Пользователь запускает ваш внешний бот.
  2. Ваш бот получает список спонсоров от Neo Link.
  3. Пользователь подписывается на каналы.
  4. Ваш бот проверяет подписки через getChatMember.
  5. Если подписка подтверждена, бот сообщает Neo Link о событии.
  6. Neo Link фиксирует подписку и учитывает доход владельца.
  7. Фоновый аудит периодически проверяет, не отписался ли пользователь.
  8. Если пользователь отписался, система делает списание автоматически.

5. Какой HTTP API нужен для реальной интеграции

Чтобы внешний бот подключался к Neo Link через интернет, нужно открыть публичные web endpoints. Ниже рекомендуемый контракт.

Получить спонсоров

POST /api/neolink/get-sponsors

{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789
}

Проверить подписку

POST /api/neolink/check-member

{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789,
  "sponsor_chat_id": "-1001234567890"
}

Зарегистрировать подтверждённую подписку

POST /api/neolink/register-subscription

{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789,
  "sponsor_chat_id": "-1001234567890",
  "sponsor_name": "Sponsor channel",
  "charge_amount": 1
}

6. Что должен делать ваш aiogram-бот

  • получать список спонсоров
  • показывать их пользователю
  • после подписки проверять статус через getChatMember
  • если статус member, administrator или creator — подтверждать подписку
  • отправлять подтверждённую подписку в Neo Link

Минимальный пример aiogram

import aiohttp
from aiogram import Bot, Dispatcher, F
from aiogram.filters import CommandStart
from aiogram.types import Message

BOT_TOKEN = "YOUR_BOT_TOKEN"
NEOLINK_API_KEY = "YOUR_NEO_LINK_API_KEY"
NEOLINK_BASE_URL = "https://your-domain.com/api/neolink"


async def neolink_request(endpoint: str, payload: dict):
    body = {"api_key": NEOLINK_API_KEY, **payload}
    async with aiohttp.ClientSession() as session:
        async with session.post(f"{NEOLINK_BASE_URL}/{endpoint}", json=body, timeout=10) as resp:
            resp.raise_for_status()
            return await resp.json()
Полный пример лежит в файле docs/neolink_aiogram_example.py.

7. Как правильно проверять подписку

Проверяйте пользователя только через Telegram Bot API, используя getChatMember.

Успешными считаются только статусы:

  • member
  • administrator
  • creator

Если пришёл статус left, kicked или Telegram API вернул ошибку, подписку подтверждать нельзя.

8. Как работает автоматическое списание

Внутри проекта есть фоновая задача, которая периодически проверяет активные подписки. Если пользователь уже не состоит в спонсоре, Neo Link:

  • помечает подписку как отписанную
  • делает списание
  • создаёт финансовую запись
  • сохраняет событие в лог
Если бот остановлен владельцем, проверки и учёт по нему временно не выполняются до повторного включения.

9. Частые ошибки и что делать

Невалидный токен

Проверьте токен у BotFather и обновите его в настройках бота.

Бот не может проверить подписку

Обычно причина в том, что бот не добавлен администратором в канал или у него нет нужных прав.

Таймаут, DNS или ошибка сети

Не считайте такую проверку успешной. Повторите запрос позже и сохраните ошибку в лог.

Одна и та же подписка подтверждается повторно

Используйте уникальную связку traffic_bot_id + user_id + sponsor_chat_id. В текущем проекте это уже предусмотрено.

10. Что нужно сделать, чтобы всё заработало end-to-end

  1. Подключить бота в Neo Link и получить API-ключ.
  2. Развернуть публичный HTTP API Neo Link.
  3. Добавить вызовы этого API в своего aiogram-бота.
  4. Проверить руками сценарии:
    • подписался
    • не подписался
    • отписался
    • бот остановлен
    • бот включён снова
