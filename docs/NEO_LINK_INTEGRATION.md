# Neo Link: полная инструкция по интеграции продажи трафика

## Что это такое

Neo Link позволяет подключить ваш Telegram-бот к системе продажи трафика.

После подключения:

- вы добавляете бота в Neo Link по токену
- получаете API-ключ
- проходите модерацию
- встраиваете в своего бота проверку спонсоров
- Neo Link начинает учитывать подписки
- при отписке пользователя система автоматически делает списание

Важно разделять две цены:

- цена продажи трафика клиенту сервиса:
  - с проверкой: `2.5 ₽`
  - без проверки: `1.5 ₽`
- доход владельца подключённого бота:
  - `1 ₽` за подтверждённую подписку

## Как подключить бота в интерфейсе Neo Link

### Шаг 1. Откройте раздел

В боте Neo Link:

- `Продажа трафика`
- `Добавить нового бота`

### Шаг 2. Подключите по токену

Система попросит:

- отправить токен Telegram-бота

После отправки токена Neo Link:

- проверит токен через `getMe`
- получит `bot_id`, `username`, `name`
- создаст запись о боте в системе
- сгенерирует `API key`
- отправит заявку на модерацию админу

### Шаг 3. Получите API-ключ

После успешного добавления бот покажет:

- ваш API-ключ
- кнопку перехода к документации
- кнопку настроек

### Шаг 4. Дождитесь модерации

Пока бот не одобрен, он находится в статусе:

- `🟡 На модерации`

После решения администратора:

- `🟢 Допущено`
- `🔴 Отклонено`

- сообщать Neo Link о подтверждённой подписке
- обрабатывать ошибки сети, таймауты и Telegram API

## Что именно должно происходить при работе

Нормальный сценарий выглядит так:

1. Пользователь запускает ваш бот.
2. Ваш бот получает список спонсоров.
3. Пользователь подписывается на нужные каналы.
4. Ваш бот вызывает проверку через `getChatMember`.
5. Если подписка подтверждена, ваш бот сообщает Neo Link о событии.
6. Neo Link фиксирует подписку и начисляет владельцу `1 ₽`.
7. Фоновый аудит периодически проверяет, не отписался ли пользователь.
8. Если пользователь отписался:
   - Neo Link фиксирует отписку
   - делает списание
   - пишет событие в логи

- либо отдельным backend-сервисом

## Рекомендуемый HTTP API для Neo Link

Ниже — рекомендуемый контракт для публичного API.

### 1. Получить спонсоров

`POST /api/neolink/get-sponsors`

Пример запроса:

```json
{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789
}
```

Пример ответа:

```json
{
  "ok": true,
  "sponsors": [
    {
      "chat_id": "-1001234567890",
      "title": "Sponsor channel",
      "link": "https://t.me/example"
    }
  ]
}
```

### 2. Проверить подписку

`POST /api/neolink/check-member`

Пример запроса:

```json
{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789,
  "sponsor_chat_id": "-1001234567890"
}
```

Пример ответа:

```json
{
  "ok": true,
  "subscribed": true,
  "status": "member"
}
```

### 3. Зарегистрировать подтверждённую подписку

`POST /api/neolink/register-subscription`

Пример запроса:

```json
{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789,
  "sponsor_chat_id": "-1001234567890",
  "sponsor_name": "Sponsor channel",
  "charge_amount": 1
}
```

Пример ответа:

```json
{
  "ok": true,
  "saved": true
}
```

### 4. Логировать подозрительную активность

`POST /api/neolink/report-suspicious`

Пример запроса:

```json
{
  "api_key": "YOUR_API_KEY",
  "user_id": 123456789,
  "reason": "too_many_checks"
}
```

## Пример интеграции для aiogram

Ниже пример минимальной схемы.

```python
import aiohttp
from aiogram import Bot, Dispatcher, F
from aiogram.filters import CommandStart
from aiogram.types import Message

BOT_TOKEN = "YOUR_BOT_TOKEN"
NEOLINK_API_KEY = "YOUR_NEO_LINK_API_KEY"
NEOLINK_BASE_URL = "https://your-domain.com/api/neolink"

bot = Bot(BOT_TOKEN)
dp = Dispatcher()


async def neolink_request(endpoint: str, payload: dict):
    body = {"api_key": NEOLINK_API_KEY, **payload}
    async with aiohttp.ClientSession() as session:
        async with session.post(f"{NEOLINK_BASE_URL}/{endpoint}", json=body, timeout=10) as resp:
            resp.raise_for_status()
            return await resp.json()


@dp.message(CommandStart())
async def start(message: Message):
    await message.answer("Бот подключён к Neo Link.")


@dp.message(F.text == "/sponsors")
async def sponsors(message: Message):
    data = await neolink_request("get-sponsors", {"user_id": message.from_user.id})
    sponsors = data.get("sponsors", [])
    if not sponsors:
        await message.answer("Сейчас спонсоров нет.")
        return

    lines = ["Доступные спонсоры:"]
    for sponsor in sponsors:
        lines.append(f"- {sponsor['title']}: {sponsor['link']}")
    await message.answer("\n".join(lines))


@dp.message(F.text.startswith("/check "))
async def check_sponsor(message: Message):
    sponsor_chat_id = message.text.split(maxsplit=1)[1]
    data = await neolink_request(
        "check-member",
        {
            "user_id": message.from_user.id,
            "sponsor_chat_id": sponsor_chat_id,
        },
    )

    if not data.get("subscribed"):
        await message.answer("Подписка не найдена.")
        return

    await neolink_request(
        "register-subscription",
        {
            "user_id": message.from_user.id,
            "sponsor_chat_id": sponsor_chat_id,
            "charge_amount": 1,
        },
    )
    await message.answer("Подписка подтверждена и сохранена.")
```

Полный пример лежит в файле:

- `docs/neolink_aiogram_example.py`

## Как проверять подписку правильно

Для проверки используйте Telegram Bot API:

- `getChatMember`

Подписка считается подтверждённой только если статус:

- `member`
- `administrator`
- `creator`

Если пришёл любой другой статус:

- `left`
- `kicked`
- ошибка доступа
- ошибка API

то подписку подтверждать нельзя.

## Как работает автосписание

В проекте есть фоновая задача, которая:

- берёт активные подписки из БД
- проверяет участника через токен подключённого бота
- если участник уже не состоит в спонсоре:
  - помечает подписку как `unsubscribed`
  - делает списание
  - записывает транзакцию
  - пишет событие в лог

Если бот переведён в состояние `Остановлен`, этот механизм по нему не работает, пока вы снова не включите бота.

## Что делать при ошибках

### Невалидный токен

Причина:

- неверный токен
- токен устарел

Что делать:

- заново получить токен у BotFather
- обновить его в настройках бота

### Бот не может проверить подписку

Причина:

- бот не админ в канале
- бот не состоит в канале
- неверный `chat_id`

Что делать:

- добавить бота администратором
- проверить права на чтение участников

### Таймаут или ошибка сети

Причина:

- Telegram API временно недоступен
- DNS ошибка
- плохое соединение

Что делать:

- повторить запрос позже
- не считать это подтверждённой подпиской
- логировать ошибку

### Повторная регистрация одной и той же подписки

Причина:

- пользователь повторно нажал кнопку проверки

Что делать:

- использовать уникальную связку:
  - `traffic_bot_id`
  - `external_user_id`
  - `sponsor_chat_id`

Именно это уже реализовано в текущей БД.

   - бот остановлен
   - бот включён снова
